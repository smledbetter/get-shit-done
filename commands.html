🌟 CORE COMMANDS 🌟

COMMAND DESCRIPTION
📌 PROJECT INITIALIZATION 📌
/gsd:new-project Initialize new project through unified flow. One command takes you from idea to ready-for-planning: deep questioning, optional domain research (4 parallel researcher agents), requirements definition with v1/v2/out-of-scope scoping, and roadmap creation. NEW!
/gsd:map-codebase Map an existing codebase for brownfield projects. Analyzes codebase with parallel Explore agents, creates 7 focused documents covering stack, architecture, structure, conventions, testing, integrations, and concerns.
📝 PHASE PLANNING 📝
/gsd:discuss-phase <number> Help articulate your vision for a phase before planning. Captures how you imagine this phase working and creates CONTEXT.md with your vision, essentials, and boundaries.
/gsd:research-phase <number> Comprehensive ecosystem research for niche/complex domains. Discovers standard stack, architecture patterns, pitfalls. Creates RESEARCH.md with "how experts build this" knowledge.
/gsd:list-phase-assumptions <number> See what Claude is planning to do before it starts. Shows Claude's intended approach for a phase so you can course-correct if Claude misunderstood your vision.
/gsd:plan-phase <number> Create detailed execution plan for a specific phase. Generates PLAN.md files with concrete, actionable tasks including verification criteria and success measures. HOT!
🚀 EXECUTION 🚀
/gsd:execute-phase <number> Execute all plans in a phase. Groups plans by wave, executes waves sequentially with plans within each wave running in parallel. Verifies phase goal after all plans complete.
/gsd:quick Execute small, ad-hoc tasks with GSD guarantees but skip optional agents. Spawns planner + executor (skips researcher, checker, verifier). Quick tasks live in .planning/quick/. NEW!
🗺 ROADMAP MANAGEMENT 🗺
/gsd:add-phase <description> Add new phase to end of current milestone. Appends to ROADMAP.md using next sequential number and updates phase directory structure.
/gsd:insert-phase <after> <desc> Insert urgent work as decimal phase between existing phases (e.g., 7.1 between 7 and 8). Useful for discovered work that must happen mid-milestone.
/gsd:remove-phase <number> Remove a future phase and renumber subsequent phases. Deletes phase directory and all references. Only works on future (unstarted) phases.
🏆 MILESTONE MANAGEMENT 🏆
/gsd:new-milestone <name> Start a new milestone through unified flow. Deep questioning, optional domain research, requirements definition, and roadmap creation. Mirrors /gsd:new-project for brownfield projects.
/gsd:complete-milestone <ver> Archive completed milestone and prepare for next version. Creates MILESTONES.md entry, archives details, creates git tag, prepares workspace for next version.
/gsd:audit-milestone Audit current milestone progress against requirements and roadmap. Shows completion status and identifies any gaps or issues.
/gsd:plan-milestone-gaps Create plans to address identified milestone gaps. Generates additional phase plans for any discovered missing work.
📊 PROGRESS TRACKING 📊
/gsd:progress Check project status and intelligently route to next action. Shows visual progress bar, summarizes recent work, displays current position, and offers to execute or create next plan. HOT!
/gsd:resume-work Resume work from previous session with full context restoration. Reads STATE.md for project context, shows current position and recent progress, offers next actions.
/gsd:pause-work Create context handoff when pausing work mid-phase. Creates .continue-here file with current state, updates STATE.md session continuity section.
/gsd:verify-work Verify completed work against plan requirements. Ensures all tasks were completed correctly and nothing was missed.
🔎 DEBUGGING 🔎
/gsd:debug [issue] Systematic debugging with persistent state across context resets. Gathers symptoms, creates .planning/debug/[slug].md to track investigation, uses scientific method. Survives /clear!
📋 TODO MANAGEMENT 📋
/gsd:add-todo [description] Capture idea or task as todo from current conversation. Extracts context, creates structured todo file in .planning/todos/pending/, infers area from file paths.
/gsd:check-todos [area] List pending todos and select one to work on. Optional area filter, loads full context for selected todo, routes to appropriate action.
🛠 UTILITY COMMANDS 🛠
/gsd:help Show complete GSD command reference. Displays all available commands with descriptions and usage examples.
/gsd:whats-new See what's changed since your installed version. Shows installed vs latest version comparison, displays changelog entries, highlights breaking changes.
/gsd:update Update GSD to the latest version. Downloads and installs the newest version of the get-shit-done-cc package.
